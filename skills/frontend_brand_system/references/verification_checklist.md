# Verification Checklist

Run this pass after planning, implementation, or review.

## Hierarchy

- Can a first-time visitor understand the page purpose in one screen?
- Is there one dominant focal point?
- Is the primary CTA obvious?
- Does the page feel direct rather than staged?

## Brand

- Does the page feel like the same OpenAI-like product family as the other projects?
- Did the page avoid generic SaaS patterns?
- Did the page avoid Anthropic/editorial energy?
- Is the brand signal strong enough without decorative branding?

## Narrative

- Does each section have one clear job?
- Is there dead weight that should be removed?
- Does the page explain too much before showing proof?
- Does the page move the user toward action quickly enough?

## Visual System

- Are typography, spacing, surfaces, and CTA states internally consistent?
- Is the palette cold-neutral rather than warm?
- Are there any serif headlines or editorial type choices that violate the mother style?
- Are cards used only where justified?
- Does the product preview carry meaning?
- Does the interface feel more like a product system than a branded campaign?

## Motion

- Are there only a few meaningful motion patterns?
- Do animations improve hierarchy or state clarity?
- Is motion still appropriate on mobile?
- Does motion stay restrained instead of theatrical?

## Responsive

- Does the first screen still work on mobile?
- Does the CTA remain visible and meaningful on mobile?
- Does the visual preview collapse in a controlled way?
- Does the page keep its OpenAI-like calm and clarity at smaller sizes?
