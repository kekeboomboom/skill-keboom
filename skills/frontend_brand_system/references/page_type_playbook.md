# Page Type Playbook

Use this file after reading the brand charter.

The default lens is OpenAI-like: calm, direct, product-led, and system-oriented.

## Landing Page

Primary job:

- establish product identity
- communicate promise
- provide proof
- move the user to the first meaningful action

Good landing pages:

- have one dominant promise
- feel like a product entry point, not a brand essay
- show proof without turning into a documentation grid
- keep the CTA obvious and immediate

Avoid:

- editorial storytelling as the primary move
- feature laundry lists
- many equal-weight sections
- warm or decorative brand packaging overwhelming product clarity

## Product Page

Primary job:

- explain a product capability or workflow in more detail
- help the user decide whether the product is relevant

Good product pages:

- explain capability layers clearly
- use real outputs and real interfaces as proof
- stay concise and product-centric
- reduce marketing packaging in favor of clarity

Avoid:

- abstract brand messaging replacing product detail
- screenshot dumps without narrative structure
- card grids doing all the explanatory work

## App Workspace

Primary job:

- help the user complete a task
- make system state obvious
- keep friction low

Good app workspaces:

- privilege the working surface
- keep input, output, and status legible
- feel like a coherent system, not a branded marketing frame
- use light containers and quiet hierarchy

Avoid:

- excessive visual chrome
- thick, layered card shells
- warm or editorial atmosphere
- burying status, billing, or auth state when it matters to task completion

## Mixed Pages

If a page mixes marketing and working surface, split the roles.

Default answer:

- homepage carries narrative and CTA
- dedicated route carries the full workspace

Only keep a light proof surface or controlled preview on the homepage. Do not wrap the homepage in large blocks of brand packaging around the workspace.
