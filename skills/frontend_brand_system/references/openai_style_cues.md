# OpenAI Style Cues

This file captures reusable OpenAI-like cues for the default mother style used by `frontend_brand_system`.

These are reference cues, not cloning instructions.

## Sources

Observed from official OpenAI properties on 2026-04-01:

- `https://chatgpt.com/`
- `https://platform.openai.com/login`

Use current official OpenAI pages as the source of truth when a fresh check is needed.

## Reusable Visual Cues

- cold white or very light gray backgrounds
- dark, high-contrast text
- modern sans-serif typography
- minimal accent color usage
- dark primary CTA with quiet secondary actions
- sparse borders and light container treatment
- almost no decorative gradients or illustration-led branding
- generous but disciplined whitespace

## Reusable Structural Cues

- one dominant focal point per screen
- direct product entry point near the center of attention
- short supporting copy
- obvious CTA hierarchy
- proof shown through real interface patterns, not abstract marketing signals

## Workspace Cues

- the interface should feel like a product system
- input and output zones should be immediately legible
- state should be quiet but clear
- containers should organize, not decorate
- interaction surfaces should feel lightweight, not card-heavy

## What To Avoid

- copying exact OpenAI layouts
- copying exact copywriting or navigation
- reproducing one OpenAI page verbatim
- treating OpenAI-like as monochrome emptiness with no product identity

## Default Rejections

- warm palettes
- serif headlines
- editorial or research-publication tone
- thick card walls
- decorative badge collections
- design gestures that overpower the product surface

## Usage Note

When the user says “like OpenAI,” interpret it by default as:

- match the calm, restrained, product-led, system-oriented feel
- do not reproduce a specific page 1:1
- allow project-specific truth and information architecture to remain intact
